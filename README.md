## Overview

EyeAI helps buying teams make faster, data-driven decisions by surfacing:

- Which SKUs need restocking **now** vs. which are overstocked
- Demand forecasts for the next 90 days per SKU
- Substitute / similar products when a SKU is out of stock
- Supplier risk scores and reorder lead-time alerts
- Social trend signals (TikTok, Instagram, Pinterest) driving demand
- AI-generated executive summaries and per-SKU buying rationale via AWS Bedrock

## Architecture

```
Browser (index.html)
       |  fetch /api/*
       v
FastAPI  (api.py)  --> AWS Bedrock (Claude Sonnet 4.6)
       |
       +-- DataService               (CSV loader / data quality)
       +-- SimilarityIndex           (cosine similarity + PCA clusters)
       +-- SupplyChainIntelligence   (risk scores, reorder engine)
       +-- CustomerSignals           (velocity, trending, returns)
       +-- RecommendationEngine      (urgency scoring, clearance alerts)
       +-- DemandForecaster          (XGBoost, weekly features)

Legacy (optional):
  Streamlit app (app.py) -- same services, port 8501
```

All data is served from synthetic CSVs (300 products, 76 k+ sales rows, 88 k+ social signals, 25 suppliers). No database required.

## Quick Start

### 1. Clone & create virtual environment

```bash
git clone <repo-url>
cd EyeAI
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r eyewear_platform/requirements.txt
```

### 3. Configure environment (optional)

Create `eyewear_platform/.env` with any overrides (see [Configuration](#configuration) below).
Without an `.env` file the app starts in **mock mode** -- no AWS credentials needed.

### 4. Start the FastAPI server

```bash
cd eyewear_platform
uvicorn api:app --host 0.0.0.0 --port 8001
```

Open **http://localhost:8001** in your browser.

> **Windows note**: Port 8000 often stays in TIME_WAIT after a previous run. Use `--port 8001` to avoid conflicts.

### 5. (Optional) Run the legacy Streamlit dashboard

```bash
cd eyewear_platform
streamlit run app.py
```

Open **http://localhost:8501** in your browser.

## Configuration

All settings live in `eyewear_platform/.env` (overrides `config/settings.py` defaults).

| Variable | Default | Description |
|---|---|---|
| `AWS_REGION` | `us-east-1` | AWS region for Bedrock |
| `AWS_BEDROCK_MODEL_ID` | `us.anthropic.claude-sonnet-4-6` | Bedrock inference profile ID |
| `AWS_ACCESS_KEY_ID` | _(empty)_ | AWS access key |
| `AWS_SECRET_ACCESS_KEY` | _(empty)_ | AWS secret key |
| `MOCK_BEDROCK` | `false` | Set `true` to disable real Bedrock calls |
| `DATA_DIR` | `data/synthetic` | Path to CSV data files |
| `MODEL_DIR` | `models/artifacts` | Path to saved model pickles |
| `FORECAST_HORIZON_DAYS` | `90` | Days ahead for demand forecast |
| `SIMILARITY_TOP_N` | `5` | Number of similar SKUs to return |
| `REORDER_SAFETY_STOCK_DAYS` | `30` | Safety stock threshold in days |
| `LOG_LEVEL` | `INFO` | Logging verbosity |

### Running without AWS

Set `MOCK_BEDROCK=true` (or omit credentials) -- all AI endpoints return realistic canned responses so every feature works locally without credentials.

## Pages & Modules

### Executive Overview
Top-line KPIs (revenue, stockouts, overstock cost, gross margin), frame-shape treemap, top-10 buy recommendations table, and an on-demand AI executive summary.

### Demand Forecasting
Select one or multiple SKUs to view historical sales vs. 90-day XGBoost forecast with confidence bands. Includes model metrics (MAPE, R2), feature importance bar chart, and demand seasonality by month.

### Product Similarity
PCA scatter plot of all SKUs coloured by cluster. Select any SKU to find its top-N similar products and identify demand spillover candidates when a SKU is out of stock.

### Supply Chain
Supplier risk table (lead time, reliability, capacity utilisation, risk band), top-20 reorder alerts, cost-saving opportunities, and a material/country concentration heat map.

### Customer Signals
Social bubble chart (velocity vs. trend score), top trending SKUs table, return-rate analysis, and age-group x frame-shape preference heat map.

### Buy Recommendations
Filterable table by urgency band (Critical / High / Medium / Low) and price point. Each row shows recommended quantity, estimated cost, margin %, supplier risk, and confidence score. Includes clearance alerts for overstocked SKUs.

### AI Buying Assistant
Multi-turn conversational assistant with full portfolio context injected. Renders responses as formatted markdown (tables, bold, lists).

### Data Quality
Per-dataset row counts, duplicate rates, null percentages, and date ranges.

| Service | Port | Description |


## Notes

- **Port conflicts on Windows**: Port 8000 frequently stays in `TIME_WAIT` state on Windows. Run the server on `--port 8001` for local development.
- **Bedrock model**: The platform targets the `us.anthropic.claude-sonnet-4-6` cross-region inference profile. Ensure it is enabled in your AWS account under **Bedrock → Model Access**.
- **Mock mode**: When `MOCK_BEDROCK=true` (the Docker default), all AI endpoints return realistic canned responses -- no AWS credentials required.
- **Data regeneration**: Run `python data/generate_synthetic_data.py` from `eyewear_platform/` to recreate all synthetic CSVs.
- **Model retraining**: Run `python pipeline.py` from `eyewear_platform/` to retrain the demand forecaster and save new pickles.
